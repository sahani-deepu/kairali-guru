# Redesign Plan

## Goal
Transform the existing website into an authentic, premium, institutionally credible Kerala Ayurveda education platform with multilingual, accessible, SEO-ready pages.

## Phase 1 — Audit and sitemap
1. Audit current homepage, header, footer, course and student hub pages.
2. Identify missing pages from requested sitemap.
3. Document missing content and feature statuses.
4. Preserve existing multilingual architecture while building new pages.

## Phase 2 — New Ayurvedic visual system
1. Define brand colour tokens and global CSS variables.
2. Introduce serif/sans typography pairing for premium editorial tone.
3. Build reusable UI components for hero, trust strip, course cards, feature panels, and location cards.
4. Add subtle botanical and manuscript-inspired details, without over-decoration.

## Phase 3 — Homepage redesign
1. Replace current homepage structure with the requested section order.
2. Build an authentic hero, trust strip, about preview, pathway selector, featured courses, heritage story, faculty preview, international learner panel, knowledge centre preview, FAQ preview, and premium footer.
3. Validate responsive layout at desktop, tablet, and mobile breakpoints.
4. Ensure page uses strong Ayurveda identity: earth tones, Kerala imagery, heritage typography, and premium spacing.

## Phase 4 — Header and footer
1. Improve header to support transparent hero state, sticky after scroll, accessible mega-menu, student login, book consultation, apply now, and language selector.
2. Build a premium footer with site navigation, course categories, contact details, language selector, policies, and verified institutional details.
3. Ensure both header and footer are consistent across all locales.

## Phase 5 — Core pages and features
1. Build /about, /courses, /courses/[slug], /student-hub, /learn, /locations, /gallery, /faq, /travel-visa.
2. Create course comparison tool, academic calendar, event pages, and student portal adapter.
3. Add consistent multilingual messages and fallback behavior.
4. Implement SEO and structured data for all indexable pages.

## Phase 6 — Site-wide relations
1. Add chatbot fallback and WhatsApp integration.
2. Add consultation booking form, exit-intent prompt, cookie consent manager, newsletter, search, and live chat fallback.
3. Ensure all forms work with client and server validation.
4. Add translations for consent manager, forms, and interface copy.

## Phase 7 — Testing and documentation
1. Build functional, accessibility, multilingual, SEO, and integration documentation.
2. Run production build and fix any console or build errors.
3. Produce final reports: functional-test-report, accessibility-report, multilingual-test-report, seo-aeo-report, integration-guide, environment-variables, content-gaps, deployment-checklist.

## Immediate next tasks
- Finalize current sitemap route table and missing pages list.
- Replace homepage with visible Ayurvedic design preview.
- Build the initial homepage sections and verify in desktop and mobile.
- Add colour tokens and typography definitions in global CSS.
